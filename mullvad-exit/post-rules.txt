# Applied by gluetun after every firewall (re)build — see README "Troubleshooting".
# Why: the tailscale sidecar (Alpine) programs iptables-LEGACY, but gluetun's
# kill-switch lives in iptables-NFT with FORWARD policy DROP — forwarded exit-node
# traffic must pass BOTH backends, so it dies there without these. Scoped to the
# tailscale0<->VPN-tunnel pair only: exit traffic can only leave via Mullvad,
# never eth0, so the kill-switch guarantee is preserved.
#
# Format rules (gluetun runUserPostRules): every rule line must start exactly with
# "iptables " at column 0, LF endings; anything else (incl. these comments) is
# silently skipped. Rules naming an absent interface are legal no-ops, so both the
# live-verified WireGuard interface (tun0, v3.41.1) and the source-default (wg0)
# are covered in case an upgrade renames it.
iptables -A FORWARD -i tailscale0 -o tun0 -j ACCEPT
iptables -A FORWARD -i tun0 -o tailscale0 -j ACCEPT
iptables -t nat -A POSTROUTING -o tun0 -j MASQUERADE
iptables -A FORWARD -i tailscale0 -o wg0 -j ACCEPT
iptables -A FORWARD -i wg0 -o tailscale0 -j ACCEPT
iptables -t nat -A POSTROUTING -o wg0 -j MASQUERADE
